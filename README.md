## Executive Summary
**NoxGuard** bridges the gap between traditional finance (TradFi) and decentralized finance (DeFi) by solving the industry's biggest bottleneck: **Financial Privacy**. 

Current Web3 lending protocols either require massive overcollateralization or force users to expose their highly sensitive financial history on a public ledger. NoxGuard utilizes the **iExec Nox Protocol** (combining Trusted Execution Environments with Fully Homomorphic Encryption) to enable autonomous, undercollateralized loans where neither the lender nor the blockchain ever sees the borrower's private data or exact credit score.

---

## 🔴 The Privacy Paradox (The Problem)
Undercollateralized lending is the holy grail of DeFi. However, the current approach requires users to expose their highly sensitive, private financial history (income, expenses, credit utilization) on a public blockchain. Institutions and retail users will never adopt Web3 lending if it means destroying their financial privacy and letting the whole world see their bank statements.

## 🟢 The NoxGuard Architecture (The Solution)
NoxGuard is an institutional-grade lending primitive that solves this by combining off-chain TEE computation with on-chain encrypted state.

1. **The Vault (TEE):** A user uploads their private financial history. Instead of going to the blockchain, it is securely routed into an air-gapped, impenetrable Intel SGX Enclave.
2. **The Blind AI:** Inside the enclave, a deterministic ONNX machine learning model evaluates the financial data, calculates a credit score, and securely deletes the raw data. No external network calls are made.
3. **The Lockbox (FHE):** The TEE cryptographically signs the score and pushes it on-chain, where it is stored as an FHE-encrypted ciphertext (`euint64`).
4. **The Magic Check:** A lender's smart contract can mathematically prove whether the user's hidden score is above their required threshold (using `TFHE.ge()`) **without ever decrypting or revealing the score**.

---

## ⚙️ iExec Nox Protocol Integration
NoxGuard heavily leverages the **iExec Nox Protocol** to achieve end-to-end privacy and composability:

- **NoxOracle:** An on-chain verifier that authenticates the ECDSA signature generated by the TEE, ensuring the AI model's output hasn't been tampered with.
- **NoxScoreToken (Confidential Token):** A Soulbound ERC-20 wrapper that stores the user's credit score as a confidential asset. It maintains hidden balances and is fully composable with existing DeFi protocols.
- **NoxCreditGate:** The state machine that manages lender access. It utilizes the `GatewayCaller` async pattern to evaluate the encrypted threshold condition and autonomously disperse funds if the criteria are met.

## 🚀 Quick Start (Local Development)

### Prerequisites
- Node.js v20+
- Hardhat
- MetaMask

### 1. Smart Contracts & Protocol Layer
```bash
npm install
npx hardhat test          # Run the 42 integration and unit tests
npx hardhat run scripts/deploy.ts --network arbitrumSepolia # Deploy infrastructure
```

### 2. Frontend Application
```bash
cd frontend
npm install
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) to view the application.

### 3. TEE Machine Learning Model (Local Execution)
```bash
cd iapp
npm install
npx ts-node src/index.ts
```

## 🛡️ Security & Privacy Guarantees
- **Zero-Knowledge Data:** Financial CSVs never touch a public server.
- **Air-Gapped AI:** The ONNX inference runs locally inside the TEE; no third-party APIs (e.g., OpenAI) are used.
- **On-Chain Cryptography:** All on-chain state comparisons are performed using Fully Homomorphic Encryption (FHE), ensuring the raw score is mathematically unreadable by nodes, miners, or lenders.
